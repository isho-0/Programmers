## Programmers C++ Solutions

이 레포지토리는 **프로그래머스(Programmers) 문제들을 C++로 디버깅하며 풀고, 그 풀이를 기록하는 개인 성장용 프로젝트**입니다.  
단순 정답 저장이 아니라, **실제로 디버깅하면서 생각 과정을 정리하는 것**을 목표로 합니다.

---

## 개발 환경

- **언어**: C++17
- **IDE**: Visual Studio 2026 (Windows)
- **프로젝트 타입**: 콘솔 애플리케이션
- **빌드 설정**: 기본 Debug/Release, Win32/x64 구성 사용

---

## 코드 구성 및 풀이 흐름

### 1. `main.cc`에서 먼저 풀기

1. 새로운 문제를 풀 때는 **항상 `main.cc`에서 직접 코드 작성**을 시작합니다.
2. 필요하면 디버거로 **입력·출력, 중간 상태를 확인**하면서 문제를 해결합니다.
3. 원하는 출력/정답이 정확히 나올 때까지 `main.cc`에서만 수정·실험합니다.

> 이 단계는 “연습장 + 디버깅용” 역할입니다.  
> 구조보다는 **정답 도달**과 **이해**에 집중합니다.

---

### 2. 정답이 나오면 라벨링: `solution_문제번호`

문제를 풀어서 **정답이 검증**되면, `main.cc` 안의 풀이 코드를 정리해서 함수로 감싸고 다음 규칙으로 라벨링합니다.

- **형식**: `solution_문제번호`
  - 예: `solution_12954`, `solution_118666` 등
- **용도**:
  - 나중에 문제별로 검색하기 쉽도록 하기 위함
  - 동일 파일 내에서 여러 문제 풀이를 함께 관리할 수 있도록 하기 위함

정리 시에는 다음을 함께 진행합니다.

- 불필요한 디버그 출력 제거
- 변수명, 들여쓰기 등 최소한의 정리
- 나중에 다른 파일로 분리하더라도 그대로 가져다 쓸 수 있는 형태로 정돈

---

### 3. 레벨별 분류 계획 (`#pragma region` + `problems/` 폴더)

추후에는 문제들을 **레벨별로 정리**합니다.

#### 3-1. `main.cc` 내부 구조

- `main.cc` 안에서 **레벨별로 문제 정의 구간을 `#pragma region` 으로 감쌀 예정**입니다.
  - 예:
    - `#pragma region LEVEL0`
    - `#pragma region LEVEL1`
  - 각 region 안에 해당 레벨의 `solution_문제번호`들을 모아둘 수 있습니다.
- `main()`에서는 테스트하고 싶은 `solution_문제번호`를 직접 호출하여 실행/디버깅합니다.

#### 3-2. 실제 파일 구조 (`problems/levelX/`)

실제 프로젝트에는 아래와 같이 **레벨별 폴더 구조**를 둡니다.

Programmers/
 ┣ main.cc
 ┗ problems/
     ┣ level0/
     ┣ level1/
     ┣ level2/
     ┗ level3/
